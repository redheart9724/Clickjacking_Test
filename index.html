🖱️ Drag-and-Drop Bypass Check manual — real gesture required

      

        
What this checks: some sites only guard against clicks. An attacker can hide a real control under an invisible layer so that dragging something across it — not clicking — triggers the action underneath.

        
Why it's manual: browsers deliberately reject scripted/fake drag events as a security measure, so this needs an actual mouse gesture from you. It can't be included in the Smart Scanner.

📖 How to use this tool


  
🔎 Smart Scanner

  
Set a target once, click "Run Full Scan," and watch the console. It checks headers (direct, then via proxy if enabled), loads the classic and nested frame tests, loads the overlay preview, and sends a postMessage probe — all automatically. If the header check gets a clear answer, the frame tests are marked automatically too; otherwise you'll be asked to confirm visually.


  
🛡️ Classic Test

  
Loads the target directly in an iframe. Look at the frame, then click "Real content" or "Blank/blocked" — cross-origin restrictions mean the browser can't tell you this programmatically, so your eyes are the verdict when the header check is inconclusive.


  
🪆 Nested-frame bypass

  
Frames the target three layers deep to check whether a client-side-only frame-busting script can be defeated by adding an extra layer of framing.


  
🎭 Overlay PoC builder

  
Drag the sliders to position a decoy button over the framed target, then "Open live PoC" to launch the exact attack page in a new tab. Click the decoy there — if it sits over real, clickable content, the overlay works. Download or copy the HTML for your report.


  
🖱️ Drag-jacking

  
Drag the chip onto the drop zone. Demonstrates that drag interactions can bypass click-only defenses. Needs a real mouse gesture — browsers block scripted drag events, so it can't be scanned automatically.


  
📡 postMessage tester

  
Load a target in the Overlay builder first. Checks whether the page's message handler responds to unsolicited, wildcard-origin messages — worth a closer look if it does.


  
📋 Header inspector

  
Now checks automatically: direct fetch first, then a chain of public CORS proxies if direct access is blocked. Paste curl output any time for a manual override.


  
⏱️ Timing test

  
Times a real two-click sequence. Fast completions flag that a confirm-flow could be vulnerable to a repositioned overlay between clicks.


  
📊 Dashboard & reports

  
Tracks every module's verdict and suggests a fix wherever something's vulnerable or worth reviewing. "Generate report image" makes a shareable PNG; the Markdown buttons make a text report.
