📖 How to use this tool


  
🎯 Target

  
Paste the URL you're authorized to test once, either here or via the Classic tab. Every module below reuses it.


  
🛡️ Classic Test

  
Loads the target directly in an iframe. Look at the frame, then click "Real content" or "Blank/blocked" — the browser can't tell you this programmatically due to cross-origin restrictions, so your eyes are the verdict.


  
🪆 Nested-frame bypass

  
Frames the target three layers deep to check whether a simple frame-busting script (one that only compares top vs self) can be defeated by adding an extra layer. Confirm the result the same way as the Classic test.


  
🎭 Overlay PoC builder

  
Drag the sliders to position a decoy button over the framed target, click "Load target", then "Open live PoC" to launch the exact attack page in a new tab. Click the decoy there — if it sits over real, clickable content, the overlay works. Download or copy the HTML for your report.


  
🖱️ Drag-jacking

  
Drag the chip onto the drop zone. This demonstrates that drag interactions can reach through a frame even when a site only guards against clicks — useful context alongside the verdict tests, not an independent pass/fail.


  
📡 postMessage tester

  
Load a target in the Overlay builder first. Sending a probe checks whether the page's message handler responds to unsolicited, wildcard-origin messages — worth a closer look if it does.


  
📋 Header inspector

  
Tries to read framing headers directly first. Most sites block this via CORS, so run the printed curl command in a terminal and paste the output back in for an automatic verdict.


  
⏱️ Timing test

  
Times a two-click sequence. A fast completion flags that a real confirm-flow could be vulnerable to a repositioned overlay between clicks.


  
📊 Dashboard & reports

  
Tracks every module's verdict in one place and suggests a fix wherever something's vulnerable or worth reviewing. Use "Generate report image" for a shareable PNG, or the Markdown buttons for a text report.
